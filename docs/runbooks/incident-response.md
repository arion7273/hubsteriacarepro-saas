# Incident response runbook

1. Triage severity, affected tenants, and whether PHI may be involved.
2. Freeze deploys unless a hotfix is required for containment.
3. Rotate exposed credentials and revoke suspicious sessions.
4. Preserve audit logs, API logs, and database snapshots.
5. Notify leadership, legal, and compliance owners.
6. Communicate to affected customers according to contractual and regulatory timelines.
7. Complete a post-incident review with corrective actions.
