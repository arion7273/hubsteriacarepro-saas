# Audit logging conventions

Audit events are append-only operational records. They should never contain raw PHI in metadata.

## Required fields

- `tenantId`: public tenant id from `x-tenant-id`.
- `actorId`: user, service, or system actor from `x-actor-id`.
- `eventType`: stable verb-noun string such as `resident.viewed`.
- `resourceType`: resource family such as `resident`, `care_plan`, or `task`.
- `resourceId`: public resource id, not an internal database UUID when avoidable.
- `requestId`: request correlation id from `x-request-id` or generated by the API.
- `occurredAt`: server timestamp in ISO-8601 format.

## Event naming

Use past-tense or access-oriented events for reads (`resident.viewed`) and action verbs for mutations (`task.updated`, `sync.applied`). Include counts, idempotency keys, and filters in metadata only when they do not expose sensitive details.
